# scipy-style-medical-image-filter-cpp
C++独立实现的医学图像滤波工具：以Scipy滤波函数为学习蓝本，复刻其核心设计逻辑（统一底层、灵活参数、滤波核计算），专注医学图像（含DICOM）的轻量化滤波操作，便于开发者学习与直接复用。

## 核心定位（学习+实用）
- 📚 学习目标：复刻Scipy滤波函数的设计思想（统一底层接口、滤波核参数化计算、边界处理逻辑），用C++独立实现（无Scipy依赖），理解滤波底层原理
- 🎯 实用价值：专注医学图像场景，提供简洁API，支持DICOM/.png/.npy等常见医学影像格式，可直接集成到科研或项目中

## 核心特性
- 🔧 纯C++实现：无Python/Scipy依赖，编译后跨平台（Windows/Linux/macOS）运行
- 📊 医学图像适配：
  - 原生支持DICOM文件读写（基于DCMTK库，保留患者信息、像素间距等医学元数据）
  - 兼容灰度医学影像（8/16位深度，适配CT/MRI等常见模态）
- 内置滤波算法（均基于统一底层函数实现）：
  - 平滑滤波：高斯滤波、均值滤波、中值滤波
  - 边缘检测：Sobel、Prewitt、拉普拉斯滤波
  - 自定义扩展：支持用户添加自定义滤波核（仅需实现核计算逻辑）
- 🎨 灵活参数配置：支持调整滤波核尺寸、sigma值、阈值、边界填充方式等，适配不同医学图像需求

## 技术栈
- 开发语言：C++17（支持现代语法，代码简洁易读）
- 依赖库：DCMTK（DICOM解析）、CMake（跨平台构建）、MinGW/GCC/Clang（编译环境）
- 无额外冗余依赖，编译后体积轻量

## 适用人群
- 🔍 学习者：想理解Scipy滤波底层原理，用C++实践的开发者/学生
- 🏥 医学影像开发者：需要轻量化滤波工具，无需重量级框架（如ITK/VTK）的科研/项目
- 代码复用者：需要简洁API，快速实现医学图像滤波的场景

## 核心亮点
- ✅ 无Scipy依赖：C++独立实现，避免Python环境限制，适合嵌入式或高性能场景
- ✅ 统一底层设计：所有滤波基于单一底层函数驱动，仅通过「滤波核计算」和「参数配置」区分，对齐Scipy设计思路
- ✅ 医学场景优化：针对医学图像高对比度、高噪声特性，优化滤波核参数默认值
- ✅ 注释详尽：核心代码含滤波原理、Scipy逻辑对比注释，便于学习复盘
